Introduction: What is HTML5?

HTML5 is the most recent version of Hypertext Markup Language used to display a web page. Combined with CSS HTML5 is very powerful because includes enhanced functionality to display animation, graphics, movies and videos without the use of a scripting language like JavaScript. Differences between HTML and HTML5

Besides using a simplier display structure, here are the key difference between HTML and HTML5, as described W3Schools.

  • New semantic elements like: <header>, <footer>, <article>, and <section>
  • New attributes of form elements like number, date, time, calendar, and range
  • New form elements: <datalist> and <output>
  • New graphic elements: <svg> and <canvas>
  • New multimedia elements: <audio> and <video>

This documentation will attempt to demonstrate some of the key differences and uses of HTML5.

New semantics for accessibility

Semantics is the study of the meanings of words and phrases in a language. Semantic elements = elements with a meaning.

A semantic element clearly describes its meaning to both the browser, developer, and visitors with visual impairments. So in HTML5 many non-semantic elements have been replaced by semantic elements.

<div> and <span> Changes to: <form>, <table>, and <article>

New elements and atrributes

HTML5 offers new elements for better document structure.

Tag Description

<article> Defines an article in a document <aside> Defines content aside from the page content <bdi> Isolates a part of text that might be formatted in a different direction from other text outside it <details> Defines additional details that the user can view or hide <dialog> Defines a dialog box or window <figcaption>Defines a caption for a <figure> element <figure> Defines self-contained content <footer> Defines a footer for a document or section <header> Defines a header for a document or section <main> Defines the main content of a document <mark> Defines marked/highlighted text <meter> Defines a scalar measurement within a known range (a gauge) <nav> Defines navigation links <progress> Represents the progress of a task <rp> Defines what to show in browsers that do not support ruby annotations <rt> Defines an explanation/pronunciation of characters (for East Asian typography) <ruby> Defines a ruby annotation (for East Asian typography) <section> Defines a section in a document <summary> Defines a visible heading for a <details> element <time> Defines a date/time <wbr> Defines a possible line-break

HTML APIs (Application Programming Interfaces) with CSS and JavaScript

Some of the new API features that HTML offers include:

  • HTML Drag and Drop

  • HTML Application Cache
  • HTML Web Workers
  • HTML SSE

Drag and drop is my favorite new feature and is very popular with many web developers. It is when you "grab" an object and drag it to a different location. In HTML5, drag and drop is part of the standard: Any element can be draggable. Below is an example of the Drag and Drop feature:

<!DOCTYPE HTML> <html> <head> <script> function allowDrop(ev) { ev.preventDefault(); } function drag(ev) { ev.dataTransfer.setData("text", ev.target.id); } function drop(ev) { ev.preventDefault(); var data = ev.dataTransfer.getData("text"); ev.target.appendChild(document.getElementById(data)); } </script> </head> <body> <div id="div1" ondrop="drop(event)" ondragover="allowDrop(event)"></div> <img id="drag1" src="../landingpage/img_logo.gif" draggable="true" ondragstart="drag(event)" width="336" height="69"> </body> </html>
